# Robot Kinematics & DH Parameters вАФ Leader & Follower

> **What this is:** DenavitвАУHartenberg (DH) parameters for the NormaCore arms, **derived** from the
> URDF files in the repo. Includes both the **exact URDF kinematics** (authoritative) and a **derived
> standard-DH table** (approximate вАФ see caveats).
> **Generated by:** `dh_from_urdf.py` (pure-Python, no deps; self-verifying). Re-runnable.

## вЪ†пЄП Read this first вАФ three honest caveats

1. **The repo contains NO DH parameters.** URDFs describe robots with link-to-link transforms +
   joint-axis vectors, *not* DH. The tables below are **derived**, not copied from NormaCore docs.
2. **The derived DH is APPROXIMATE.** These CAD-exported arms have (a) **near-parallel consecutive
   axes** (the DH offset `d` is mathematically ambiguous there) and (b) **slightly skew/tilted axes**
   (~0.5вАУ1¬∞). A single DH transform can't perfectly represent skew joints вАФ the "fit residual" column
   reaches ~0.09 on those joints. **For real IK/FK, use the URDF directly** (`ikpy`/PyBullet load it
   exactly); DH here is for reference/pitch/tools that require DH input.
3. **Leader vs follower:**
   - **ElRobot:** only a *follower* URDF exists in the repo. The **leader is the same arm geometry**
     (difference is motors: leader = 7.4 V back-drivable, follower = 12 V high-torque) вЖТ **identical
     kinematics/DH.**
   - **SO-101:** separate leader & follower URDFs exist, but they produce **identical DH** (verified) вАФ
     same joint chain; only a tiny wrist/handle offset differs.

## DH convention used

Standard (distal) DenavitвАУHartenberg:
`A_i = Rotz(ќЄ_i) ¬Ј Transz(d_i) ¬Ј Transx(a_i) ¬Ј Rotx(ќ±_i)`
The `ќЄ_offset` below is the constant offset at the home configuration; the **commanded joint angle adds
to it**. Lengths in metres, angles shown in both rad and deg.

## How to reproduce / use

```bash
python3 dh_from_urdf.py        # prints exact kinematics + DH for all three URDFs, with residuals
```

**Recommendation for our project:** for IK/FK, **load the URDF directly** (see
[`06-feasibility-and-decisions.md`](./06-feasibility-and-decisions.md) D3) вАФ `ikpy`/PyBullet use the
exact model and avoid the DH approximation entirely. Use the DH tables above only if a specific tool
requires DH input, and remember rows marked вЪ†пЄП are nominal.
